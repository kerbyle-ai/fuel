# Доступ к Timeweb VPS без SSH (kvmnvm-939)

**Сервер:** kvmnvm-939 · **IP:** 147.45.175.194 · **Проект:** fuel-map

Симптом: `ssh root@147.45.175.194` → `Connection timed out`. В **серийной** консоли панели: «выполните `serial_enable.sh`» — замкнутый круг, если SSH недоступен.

> **Важно:** `serial_enable.sh` нужен только для **серийной** консоли. **VNC** и **режим восстановления** работают отдельно и не требуют этого скрипта заранее.

## Вариант B: Облачный файрвол (уровень Timeweb, не ufw)

Это **FWaaS** — правила в панели, не на сервере. `ufw` на VPS можно не трогать, пока не появится консоль.

### Путь в панели (UI 2024/2025)

1. Войти: [timeweb.cloud](https://timeweb.cloud/) → проект с сервером.
2. Боковое меню: **Сети** → **Firewall** (или **Файрвол**).
   - Альтернатива: карточка сервера → вкладка **Сеть** → блок **Группы безопасности / Firewall**.
3. **Создать группу** (тип: **Разрешающая / whitelist**).
4. Добавить **входящие** правила:

   | Протокол | Порт | Источник | Действие |
   |----------|------|----------|----------|
   | TCP | 22 | 0.0.0.0/0 | Разрешить |
   | TCP | 80 | 0.0.0.0/0 | Разрешить |
   | TCP | 443 | 0.0.0.0/0 | Разрешить |
   | TCP | 8090 | 0.0.0.0/0 | Разрешить (тест fuel-map) |
   | ICMP | — | 0.0.0.0/0 | Разрешить (опционально, ping) |

5. **Применить группу** к серверу **kvmnvm-939** (кнопка «Добавить сервис» / «Связать»).
6. Убедиться, что сервер **не** в **запрещающей (blacklist)** группе с правилом «всё».
7. Подождать 1–2 минуты → проверить SSH (VPN выключен).

### Как выглядит в UI (описание «скриншота»)

- Слева список групп файрвола, справа таблица правил.
- Колонки: **Направление** (Входящий), **Протокол**, **Порт**, **CIDR**, **Описание**.
- На карточке сервера во вкладке **Сеть** видна привязанная группа; если пусто — облачный файрвол **не фильтрует** (тогда проблема не в FWaaS).

Документация: [Управление файрволом](https://timeweb.cloud/docs/firewall/upravlenie-fajrvolom).

## Вариант C: Тест с мобильного хотспота

Цель: отделить проблему сервера от VPN/провайдера на ПК.

1. На телефоне: **Режим модема / точка доступа Wi‑Fi**.
2. Подключить ПК к этой сети.
3. **Полностью выключить VPN** (sing-tun и любые другие).
4. PowerShell:

```powershell

ssh-keygen -R 147.45.175.194
```

5. Интерпретация:
   - **С хотспота работает, с домашней сети нет** → роутер, VPN или провайдер; не сервер.
   - **Нигде не работает** → облачный файрвол (B) или сервер; VNC/Rescue (D/E).

## Вариант D: VNC-консоль (без serial_enable.sh)

Серийная и VNC — **разные вкладки**. Сообщение про `serial_enable.sh` относится к **Серийной**, не к VNC.

### Путь в панели

1. **Облачные серверы** → **kvmnvm-939**.
2. Вкладка **Консоль**.
3. Переключатель: выбрать **VNC** (не «Серийная»).
4. **Дашборд** → скопировать **root-пароль**.
5. В VNC: логин `root`, пароль — **Ctrl+Shift+V** (вставка в веб-консоли Timeweb).
6. Если экран «залип» — кнопка **«Вставить из буфера»** над окном консоли.

### Команды в VNC (вставить целиком)

Минимум — serial + SSH:

```bash
wget -O - https://st.timeweb.com/cloud-static/scripts/serial_enable.sh | bash
systemctl restart ssh
ufw allow 22/tcp 2>/dev/null; ufw allow 8090/tcp 2>/dev/null; ufw reload 2>/dev/null
```

Полный bootstrap fuel-map — скрипт из репозитория:

```bash
# Скопируйте содержимое deploy/vnc-one-shot.sh из репозитория
```

Или по шагам после входа в root:

```bash
bash /opt/fuel-map/deploy/vnc-one-shot.sh
```

(если репозиторий уже на диске; иначе — вставить `vnc-one-shot.sh` из `deploy/vnc-one-shot.sh`).

После `serial_enable.sh` → **перезагрузка** в панели → станет доступна **серийная** консоль (удобнее копипаст).

Документация: [Веб-консоль](https://timeweb.cloud/docs/cloud-servers/manage-servers/console).

## Вариант E: Режим восстановления (Rescue / SystemRescue)

**Без переустановки ОС.** Загрузка с LiveCD Timeweb; диск сервера монтируется снаружи.

### Путь в панели

1. **Облачные серверы** → **kvmnvm-939** → вкладка **Доступ**.
2. Блок **Режим загрузки ОС**.
3. Выбрать **Режим восстановления** (LiveCD SystemRescue).
4. **Сохранить и перезагрузить**.
5. Вкладка **Консоль** → VNC → дождаться приглашения shell.
6. После работы **обязательно** вернуть **Стандартный режим загрузки** и снова **Сохранить и перезагрузить**.

Документация: [Смена режима загрузки](https://timeweb.cloud/docs/cloud-servers/manage-servers/boot-mode), [Сброс пароля root](https://timeweb.cloud/docs/unix-guides/root-password-reset).

### Команды в Rescue (починка SSH + serial)

```bash
# Попробовать vda1, если ошибка — sda1
mount -o rw /dev/vda1 /mnt || mount -o rw /dev/sda1 /mnt

# Войти в установленную систему
chroot /mnt /bin/bash

# Внутри chroot:
wget -O - https://st.timeweb.com/cloud-static/scripts/serial_enable.sh | bash
apt-get update -qq && apt-get install -y -qq openssh-server
systemctl enable ssh
passwd root   # задать новый пароль, если забыли

# Отключить блокирующий ufw на время диагностики
ufw disable || true

# Проверить, что sshd слушает 22
ss -tlnp | grep ':22'

exit
umount /mnt
```

7. Панель → **Доступ** → **Стандартный режим** → перезагрузка.
8. SSH с ПК (хотспот, VPN выкл.).

## Вариант A: Переустановка Ubuntu + cloud-init

**Крайний вариант.** Все данные на системном диске будут удалены (снимки/бэкапы в панели сохраняются отдельно).

### Путь в панели

1. **Облачные серверы** → **kvmnvm-939** → **Конфигурация**.
2. Кнопка **Переустановить**.
3. **ПО:** Ubuntu 22.04 LTS (чистая ОС).
4. **SSH-ключ:** добавить ключ из `deploy/ssh/sap-ws01.pub` (или выбрать сохранённый).
5. Блок **Cloud-init** → **Редактировать** → вставить содержимое:
   - **YAML:** `deploy/cloud-init.yaml` или `deploy/cloud-init-timeweb.txt` — **первая строка обязательно `#cloud-config`** (без пустых строк и комментариев выше).
   - **Shell:** `deploy/cloud-init-oneliner.sh` — **первая строка обязательно `#!/bin/sh`**.
   Иначе панель Timeweb отклонит user-data.
6. **Переустановить** → подтвердить имя сервера `kvmnvm-939`.
7. Дождаться статуса **В сети** (5–15 мин).
8. Проверка:

```powershell
ssh -i $env:USERPROFILE\.ssh\id_ed25519 root@147.45.175.194
# или с паролем из панели
```

9. Загрузить код приложения (с ПК, WSL):

```bash
cd /path/to/fuel-map
SSHPASS='пароль-root' ./deploy/deploy-vps.sh
```

### Требование Timeweb к первой строке user-data

Панель принимает cloud-init только если **самая первая строка** файла — один из двух заголовков:

| Формат | Первая строка | Файл в репозитории |
|--------|---------------|-------------------|
| YAML cloud-config | `#cloud-config` | `deploy/cloud-init.yaml`, `deploy/cloud-init-timeweb.txt` |
| Shell script | `#!/bin/sh` | `deploy/cloud-init-oneliner.sh` |

Пустая строка или комментарий **до** заголовка — ошибка. Копируйте файл целиком, начиная с первой строки.

### Что делает cloud-init из репозитория

- `serial_enable.sh` — серийная консоль в панели.
- Docker + `systemctl enable docker`.
- `mkdir -p /opt/fuel-map` (placeholder; **полный код приложения** — отдельно по SSH или вручную).
- UFW: 22, 8090.
- SSH-ключ SAP-WS01 в `authorized_keys`.
- Лог: `/var/log/fuel-map-cloud-init.log`.

Документация: [Переустановка ОС](https://timeweb.cloud/docs/cloud-servers/manage-servers/os-reinstall), [Cloud-init](https://timeweb.cloud/docs/cloud-servers/manage-servers/cloud-init).

## Cloud-init на УЖЕ работающем сервере (ловушка)

Если сервер **не переустанавливать**:

| Действие в панели | Сработает без доступа к серверу? |
|-------------------|----------------------------------|
| Конфигурация → Cloud-init → Редактировать + **обычная перезагрузка** | **Нет** — `runcmd` выполняется один раз; нужно `cloud-init clean --reboot` **на сервере** |
| Cloud-init при **переустановке** | **Да** |
| Cloud-init при **создании** нового сервера (шаг 7 мастера) | **Да** |

Путь редактирования на живом сервере (когда появится консоль/SSH):

1. **Конфигурация** → блок **Cloud-init** → **Редактировать** → сохранить.
2. На сервере: `cloud-init clean --reboot`.

Для `bootcmd` (каждая загрузка) — только через YAML `#cloud-config`; shell-скрипт в панели превращается в одноразовый `runcmd`.

Мастер создания: **Облачные серверы** → **Создать** → шаг **7. Сценарий cloud-init**.

## Что делает serial_enable.sh

Официальный скрипт Timeweb (`https://st.timeweb.com/cloud-static/scripts/serial_enable.sh`):

- Проверяет `/dev/ttyS0` и параметр ядра `console=ttyS0`.
- Настраивает GRUB/SYSLINUX для вывода на серийный порт.
- Включает `serial-getty@ttyS0` (или аналог).
- Нужен для серверов, созданных **до 4 апреля 2024**, чтобы работала **серийная** консоль в панели.

**Не блокирует SSH.** SSH и VNC от него не зависят.
