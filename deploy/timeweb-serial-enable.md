# Включение серийной консоли Timeweb Cloud (VPS fuel-map)

Пошаговая инструкция для сервера **147.45.175.194** (kvmnvm-939, проект fuel-map).

Поддержка Timeweb сообщает: пункт **«Серийная консоль»** в панели появляется только после выполнения на **самой виртуальной машине** официального скрипта.

---

## Перед началом

1. **Отключите VPN** на компьютере, с которого потом будете подключаться по SSH (VPN часто даёт обрыв на этапе SSH banner).
2. Подготовьте доступ к панели [Timeweb Cloud](https://timeweb.cloud): логин, сервер **kvmnvm-939**.
3. Имейте под рукой пароль 
oot из панели (раздел сервера → доступ).

---

## Шаг 1. Понять, зачем нужен скрипт

Серийная консоль — это доступ к терминалу Linux **мимо SSH** (через гипервизор). Он нужен, когда:

- SSH обрывается (anner exchange, timeout);
- нужно починить sshd, диск, Docker;
- нужно задеплоить fuel-map в /opt/fuel-map без рабочего SSH.

Скрипт Timeweb настраивает:

- вывод ядра на 	tyS0 (115200);
- GRUB / загрузчик;
- сервис serial-getty@ttyS0;
- после настройки требуется **перезагрузка**.

Команда от поддержки:

`ash
wget -O - https://st.timeweb.com/cloud-static/scripts/serial_enable.sh | bash
`

**С Windows эту команду на VPS не «дотянуть»** — её нужно выполнить **внутри Linux на сервере** (см. шаги ниже).

---

## Шаг 2. Как попасть на сервер без SSH

Выберите один из вариантов (по приоритету):

### Вариант A — уже есть VNC в панели

1. Timeweb Cloud → **Серверы** → ваш VPS → **Консоль** / **VNC**.
2. Войдите: 
oot + пароль из панели.
3. Перейдите к **шагу 3**.

### Вариант B — серийная консоль ещё «не поддерживается»

1. Если в панели только сообщение про скрипт — нужен **другой канал**:
   - **переустановка ОС** с **cloud-init** / «скрипт при первом запуске» (шаг 4);
   - **тикет в поддержку**: попросите выполнить serial_enable.sh на вашем VPS или включить serial с их стороны.

### Вариант C — SSH вдруг заработал

`ash
ssh -o ConnectTimeout=60 -o ServerAliveInterval=5 root@147.45.175.194
`

Если сессия открылась — сразу **шаг 3** (VPN должен быть выключен).

---

## Шаг 3. Запуск скрипта на VPS

В консоли на сервере (root):

`ash
wget -O - https://st.timeweb.com/cloud-static/scripts/serial_enable.sh | bash
`

Или эквивалент:

`ash
curl -fsSL https://st.timeweb.com/cloud-static/scripts/serial_enable.sh | bash
`

Что произойдёт:

1. Скрипт проверит, что вы **root**.
2. Запишет подробный лог в /var/log/serial_setup.log.
3. Настроит GRUB и getty для **ttyS0**.
4. Спросит: **перезагрузить сейчас?** — ответьте y (рекомендуется).

После reboot подождите 2–5 минут.

---

## Шаг 4. Cloud-init (если консоли нет совсем)

При **переустановке** Ubuntu 22.04 в панели Timeweb, если есть поле **cloud-init** / user-data, вставьте:

`yaml
#cloud-config
runcmd:
  - curl -fsSL https://st.timeweb.com/cloud-static/scripts/serial_enable.sh | bash
`

Либо одной строкой в «скрипт при старте» (синтаксис зависит от UI):

`ash
curl -fsSL https://st.timeweb.com/cloud-static/scripts/serial_enable.sh | bash
`

**Внимание:** переустановка сотрёт данные на диске, если не сохранили бэкап /opt/fuel-map.

---

## Шаг 5. Открыть серийную консоль в панели

1. Timeweb Cloud → VPS → **Серийная консоль** (после enable + reboot).
2. Должен появиться приглашение login — введите 
oot и пароль.
3. Проверка:

`ash
grep console=ttyS0 /proc/cmdline
systemctl is-active serial-getty@ttyS0.service
tail -50 /var/log/serial_setup.log
`

---

## Шаг 6. Восстановление сервиса и деплой fuel-map

В серийной (или VNC) консоли:

`ash
# диагностика (из репозитория на сервере или вставить вручную)
bash /opt/fuel-map/deploy/vnc-one-shot.sh
`

Если каталога ещё нет — загрузите архив через панель (файловый менеджер / SFTP, когда SSH появится) или склонируйте репозиторий.

Целевой путь: **/opt/fuel-map**.

После стабилизации SSH (без VPN):

`powershell
# с Windows, из каталога проекта
wsl bash deploy/deploy-vps.sh
`

Подробнее: docs/DEPLOY-TIMEWEB.md, deploy/TIMEWEB-SSH-FIX.md.

---

## Шаг 7. Если что-то пошло не так

| Симптом | Действие |
|---------|----------|
| В панели снова «консоль не поддерживается» | Тикет в поддержку + приложите вывод /var/log/serial_setup.log (скопируйте из VNC/serial). |
| Скрипт упал на GRUB | Убедитесь, что ОС Ubuntu/Debian; при нестандартном образе — просите поддержку настроить serial вручную. |
| SSH всё ещё timeout | В serial: uptime, df -h, systemctl restart ssh, проверьте UFW и нагрузку (см. deploy/TIMEWEB-SSH-FIX.md). |
| Нужен деплой без SSH | Только через консоль + ручная распаковка архива или тикет Timeweb на загрузку в /opt/fuel-map. |
